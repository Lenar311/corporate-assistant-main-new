# 🔍 Поиск по нормативным документам (RAG)

![Python](https://img.shields.io/badge/Python-3.12-blue?logo=python&logoColor=white)
![Ollama](https://img.shields.io/badge/Ollama-supported-green?logo=llama&logoColor=white)
![ChromaDB](https://img.shields.io/badge/Vector%20DB-ChromaDB-orange)


Локальная система поиска по нормативным документам (ГОСТ, ОСТ, СП, СНиП, СанПиН и др.) с использованием RAG (Retrieval-Augmented Generation). Индексация документов + семантический поиск + генерация ответов через LLM. Все данные обрабатываются локально.

---

## 🏗️ Архитектура
Документы (PDF/DOCX) → Извлечение текста → Chunking → Эмбеддинги → ChromaDB
↓ ↓
pdfminer / Vision-модель Релевантные фрагменты
↓
Пользователь → Запрос → Эмбеддинг → Поиск в ChromaDB → Контекст → LLM → Ответ

🔧 Возможности
Поддерживаемые форматы
PDF — текстовые (pdfminer) и сканированные (vision-модель Ollama)

DOCX — Microsoft Word с извлечением таблиц

Автоматическое распознавание стандартов
Система автоматически извлекает метаданные из документов:

ГОСТ, ГОСТ Р ИСО — межгосударственные стандарты

ОСТ — отраслевые стандарты

СП — своды правил

СНиП — строительные нормы и правила

СанПиН — санитарные правила и нормы

ТУ — технические условия

РД — руководящие документы

ЕСКД/СПДС — единые системы конструкторской/проектной документации

Оптимизации
Инкрементальная индексация: индексируются только новые/изменённые файлы (отслеживание через SHA-256)

Кэширование поисковых запросов: до 128 уникальных запросов

Автоматическое определение GPU/CPU: ускорение на видеокартах NVIDIA

Семантическое разбиение: чанки создаются с учётом структуры документа (разделы, главы)

Извлечение таблиц: данные из таблиц DOCX включаются в индекс

📋 Требования
Python 3.8+

Ollama (для LLM и vision-моделей)

CUDA (опционально, для ускорения на GPU)

⚠️ Ограничения
Vision-модель для OCR работает медленнее Tesseract, но даёт более качественные результаты для документов со сложной структурой (таблицы, схемы)

Размер контекста LLM ограничен (используются только TOP_K наиболее релевантных фрагментов)

### 🧠 `rag_chain.py` — ядро RAG-системы

**Назначение**: Поиск и генерация ответов на вопросы по нормативным документам (ГОСТ) с использованием RAG (Retrieval-Augmented Generation).

**Ключевые функции**:
- 🔍 **Умный поиск** в векторной базе ChromaDB с фильтрацией по номеру ГОСТ и пункту
- 🎯 **Двойной поиск** при запросе конкретного пункта документа
- 🔄 **Расширение запросов**: автоматическая замена аббревиатур (КД → конструкторская документация) и добавление синонимов
- 🤖 **Генерация ответов** через Ollama с системным промптом для нормоконтроля
- 🛡️ **Fallback-механизм**: возврат релевантного фрагмента, если LLM не смог сгенерировать ответ
- 📋 **Извлечение метаданных**: автоматическое определение номера ГОСТ и пункта из текста
